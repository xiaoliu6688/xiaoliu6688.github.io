Awk处理文本数据

格式：

awk [选项] ‘匹配规则和处理规则’  [处理文本路径]

  • 处理规则：

    BEGIN{}: BEGIN是在awk处理文本之前运行

    / / ： 使用的匹配原则

    {}：循环（每次处理一行）

    END{}: 当所有行执行完毕后，执行END的操作

原理：

  1. awk会接收一行作为输入，将这一行赋给awk的内部变量$0，行的边界是以换行符作为结束的。

  2. 刚刚读入的行被以:为分隔符分解成若干字段（或域），每个字段存储在已编号的变量中，编号从$1开始，最多达100个字段，-F 指定分隔符，默认是空格。

  3. 使用print打印，如果$1$3之间没有逗号，输出时贴在一起，如果添加”,”，$1和$3之间默认用空格分开，通过OFS修改输出的间隔符 ‘BEGIN{OFS=”-“}{print $1,$3}’

  4. 输出后，获取下一行，执行上述操作

awk变量
$0:当前行的内容
NR：行号
NF: 总字段数
FS：输入字段分隔符，默认空格
OFS：输出字段分隔符
awk -F: '{if(NR > 7){print NR,$0}}' /etc/passwd
== awk -F: 'NR > 7 {print NR,$0}' /etc/passwd
// 输出第7行之后的内容

指定','作为分隔符，-F','
-F:  ：表示以':'作为分隔符
awk -F':' 'BEGIN{FS=","}{print $1,$2}' 
FS的优先级大于-F指定的

输出指定分隔符
awk -F':' 'BEGIN{OFS=" >>> "}{print $1,$2}'
  • 正则表达式
awk -F: '/^xxx/' /etc/passwd
  • 条件模式
awk -F: 'NR > 7' /etc/passwd
  • 算数运算
awk -F: '{print $1,$2+1}'
  • if判断
单分支：
	if(){}
双分支：
	if(){} else {}
多分支：
	if(){} else if(){} else {}
ls * | uniq   			剔除重复行
ls * | sort -u

Sed处理文本数据

非交互式的，流编译器sed。

sed [options] "command" files

sed和grep不同，不管是否找到匹配的模式，退出都返回0.只有命令存在语法错误时返回1.
sed在文件查找模式中时，也可以使用正则表达式RE和各种元字符。正则表达式在斜杠//之间。

扩展元字符集： ？，+，{}，|，()

选项
[-r] 支持扩展元字符
[-i] 修改源文件
[-n] 取消默认输出
命令
删除
sed -r "3{d;}" filename 				# ;用来隔开命令 
sed -r "/root/{n;d}" filename
替换
sed -r "s/root/alice/gi" filename		# g:global, i:ingore 大小写
sed -r "s/[0-9]\{2\}$/#&/" filename		# &表示查找串中匹配到的内容  
sed -r "s/root/&LIU/g"	filename		# 在root后面加LIU
sed -r "s/^/#/"	filename 				# 在开头加 #

sed -r "/root/{n;s/bin/uuu/g}" filename
查找
sed -r "/root/d" filename		#d 删除
追加
sed -r "2a111111" filename 		#第二行后面追加111111 
sed -r "2i111111" filename 		#第二行前面追加111111 
sed -r "2c111111" filename 		#第二行替换·111111 
sed -r "2a111111\
		111111"					#第二行追加多行
反向选择
sed -r "3d" filename 
sed -r "3!d" filename
修改文件
sed -ri "$a123456789" filename	
sed -ri "/^SELINUX=/cSELINUX=disaled" filename  	#查找SELINUX开头的行，并替换

sed -r "2,6s/.*/#$/" filename	 添加注释
sed -r "2,6s/(.*)/#\1/" filename 添加注释
sed -r "1,5s/^#*/#/" filename	 添加注释
地址
决定对哪些行处理
sed -r "/root/,5d" filename		#root后，删除到第5行 
sed -r "/root/,+5d" filename  	#root后，删除5行
sed -r "/root/!5d" filename  	#取反删除
sed -r "1,4d" filename
sed -r "1~2d" filename 			#删除奇数行
sed -r "0~2d" filename 			#删除偶数行	以0行开始，间隔2行删除。

模式空间和暂存空间

小写g和h是覆盖。小写G和H是追加。 暂存空间只有一个。

sed -r "1h;$G" filename
sed -r "1h;2,$g" filename

正则表达式RE

处理文本，做匹配。正则表达式放在两个双斜杠之间//

元字符

^ ：匹配行开头
[0-9]：匹配数字
+：至少一个
$：匹配行结尾
. ：匹配任意单个字符
*：前导符出现0次或多次，*b => b, bb
.*：任意多个字符
[]：匹配指定范围内的一个字符，[Aa]
[ - ]：匹配指定范围内的一个字符，[a-z]
[ ^ ]：匹配不在指定范围内的一个字符，[^a-z]
\：转义元字符
\<：词首定位符
\>：词尾定位符
x\{m\}：字符重复出现m次 o\{5\}
x\{m,\}：字符重复出现m次以上
x\{m,n\}：字符重复出现（m,n）次
\(..\) ：后面匹配时所用到的标签 %s#\(10.2\).3#\1.4#

扩展正则表达式元字符

+：匹配一个或者多个前导字符
？：匹配零个或一个前导字符
a|b：匹配a或b
()：组字符

shell中的通配符 * ：任意多个字符，由shell来解析；shell中的.相当于正则中的?，匹配单个字符。
正则表达式元字符：由各种模式匹配操作的程序来解析，比如grep，sed，awk

grep中，匹配模式在“ ”之间指出，可以匹配多个文件。

grep 文本处理

grep [选项] PATTERN filename1 filename2 grep “\-v” filename
打印出匹配模式的行

返回值：0表示找到，1表示没找到，2表示找不到指定的文件

ps aux| grep “sshd”

\w 所有字母和数字 [0-9a-zA-Z]
\W 所有字母和数字之外的字符，非字符
\b 词边界 \< => \b

[-v] 取反
[-q] 静默
[-i] 忽略大小写
[-r] 递归文件夹的文件
[-o] 只显示匹配的内容
[-c] 匹配的上下文两行
[-a] 匹配的后两行
[-b] 匹配的前两行
