如何提高仿真器的效率

了解仿真器的底层原理，合适的code style会节省仿真时间。

仿真算法：

  1. 基于时间的（spice仿真器）

    每个时间节点对所有元件都进行计算，效率低。

  2. 基于事件的（Verilog XL，NC verilog）

    只有电路发生变化时，才进行处理。效率高。

  3. 基于周期的（Verilator ）

    只在时钟边沿计算，不管时序，使用两值逻辑。仅适用于同步电路，效率高。比如CPU的仿真。

​ 以下的测试都是基于Verilog-XL

  • case语句和多if else if语句

    // case                         // if else-if                    
always @（*）begin                always @（*）begin                
    case(sel[1:0])                  if(sel == 2'b00)             
        2'b00: out = a;                 out = a;                
        2'b01: out = b;             else if(sel == 2'b01)             
        2'b10: out = c;                 out = b;                
        2'b11: out = d;             else if(sel == 2'b10)             
        default : out = a;              out = c;                
    endcase                         else if(sel == 2'b11)             
end                                     out = d;                
                                   end

    测试案例：8选1数据选择器，

    仿真发现：基于case的多路选择器，CPU仿真时间更短。

  • 带有begin end和不带begin end

    always @(posedge clk or negedge nrst)
begin
    if(a) c <= b;
    else  c <= d;
end

always @(posedge clk or negedge nrst)
    if(a) c <= b;
    else  c <= d;

​ 测试结果：nobegin表述的内存利用率和仿真时间要优一些。

  • `define 和 parameters

​ parameters要更好一些。

  • always group和nogroup

    always @(posedge clk or negedge nrst) begin
    if(en) begin
        out1 <= a;
        out2 <= b;
        out3 <= c;
        out4 <= d;
    end
end
always @(posedge clk or negedge nrst) begin
    if(en) out1 <= a;
end
always @(posedge clk or negedge nrst) begin
    if(en) out2 <= b;
end
always @(posedge clk or negedge nrst) begin
    if(en) out3 <= c;
end
always @(posedge clk or negedge nrst) begin
    if(en) out4 <= d;
end

    group的仿真时间明显缩短了。

  • verilog可综合

​ 左移，右移逻辑的操作数，可以是常数，也可以是变量，都是可以综合的，综合结果不一样。 常数的操作数，综合 出来是一个定值，变量的操作数综合出来是一个mux。

  • DC时综合的原语
// synopsys translate_off
忽略该部分的代码，DC时不会进行转移。
// synopsys translate_on
忽略的部分在前仿时是可以发挥作用的，如果错误使用translate_off on语句，会导致前仿和后仿的结果不一致。

// 小心使用translate off on
ex.带两个异步复位信号的D触发器
一般的实现方式如下：
always @(posedge clk or negedge nrst or negedge nset) begin
    if(!nrst) // asyn
        q <= 0;
    else if(!nset) // asyn
        q <= 1;
    else 
        q <= d;
end
这种方式会导致前仿和后仿出现细微的区别。极端测试向量如下：
如果nrst和nset同时拉低，则结果输出为0.经过半个周期后，nrst拉高了，输出结果依然为0.因为always块是边沿敏感触发的，但是nset和nrst都是异步的，nset此时为0，q输出结果应该为1；计算结果与实际电路不符。
解决办法：
    加入translate on off语句。配合force 和 relase
正确code如下：
always @(posedge clk or negedge nrst or negedge nset) begin
    if(!nrst) // asyn
        q <= 0;
    else if(!nset) // asyn
        q <= 1;
    else 
        q <= d;
end
// synopsys translate_off
always @(nrst or nset) begin
    if(nrst && !nset)   force q = 1;
    else                release q;
end
// synopsys translate_on
该代码可以保证，在nrst撤离时，nset如果为0，q输出结果为1；这是在前仿时需要加入的特殊语句，并加上原语。在实际综合过程中，不会被综合。保证了前仿和实际流片的结果完全一致。

在工艺库中，有时钟，异步复位和异步置数的端口，也就是说always块中的敏感列表，最多可以有三个。但是不能是同时posedge clk or negedge clk。一个信号不能依赖两个边沿。

  • verilog调试

    bg：verilog中的参数可以用 `define和parameter指定，这种方式必须要求我们在编译前修改好，编译后就不能再修改了。但是为了节省编译时间，可以使用 $test$plusargs和$value$plusargs来指定。

`timescale 1ns/1ps
module tb;
reg[255:0] testname;
initial begin
$display("**********************RESULT IS BELOW*********************");
if($test$plusargs("TAISHAN"))
$display("****************TAISHAN is selected******************");
else
$display("**********************$test$plusargs FAIL**************");
if ($value$plusargs("TESTNAME=%s", testname[255:0])) 
 begin 
 $display("*****************Running test %0s.********************", testname); 
 end
 else
 $display("**********************$value$plusargs FAIL**************");
$finish;
end
endmodule

使用方式：
1.	simv
2.	simv+TATSHAN
3.	simv+TAISHAN+TESTNAME=HUASHAN

在仿真时，0时刻会执行一些系统函数，有些数据没有准备好，导致发生系统仿真错误，因此，可以在代码中初始化变量的值。
